# 示例：对公授信预尽调

> 完整可参考的种子模板：金融 / 银行业务 / 预尽调场景。已在生产环境跑过 4 个真实主体（万科 / 宁德时代 / 海底捞 / 大疆）通过 8 项验收，可直接 `+create` 或在此基础上 `+author` 拷贝裁剪。

业务背景：客户经理拿到一个待授信企业，需要在 30 分钟内汇集公开监管披露、司法记录、舆情动向与行业景气，整理成一份可签字、可回查、可被授信委复议的预尽调底稿。

### `search_plan`

```markdown
**[[主体核验] 公开身份信息]**
- **数据路由**: 工商登记 / A 股年报与公告 / 港股披露易 / 海外母公司 SEC 文件 / 非上市企业的纳税信用与监管 ID
- **执行动作**: 法定代表人、注册资本、股权穿透至自然人或国有资本、上市状态、最新一期审计意见、年报披露日；非上市企业取最新纳税信用等级与许可证状态
- **验证策略**: 工商与披露不一致时以披露为准并标注差异；未披露字段标 "待补充"，禁止编造

**[[财务实证] 三年财务与审计意见]**
- **数据路由**: A 股 / 港股的最近 3 个完整会计年度财报 / 评级机构最新评级报告 / 行业基准数据
- **执行动作**: 拉取近三个会计年度营收/净利润/经营性现金流/总资产/负债率，核对审计意见（无保留/保留/否定/无法表示）；评级有变化的拉评级窗口；非上市企业以同业可比 + 行业基准做估算区间并明确标注
- **验证策略**: 财报与评级口径冲突以评级机构最新报告为准；财报缺失不编造，改用行业基准估算 + 明示假设；财务异常波动（>20%）必须给可能归因或标"待解释"

**[[行业景气] 监管与行业风向]**
- **数据路由**: 行业协会 / 证监会-金融监管总局公告 / 海外监管文件（SEC / EU / OFAC） / 主流行业研报
- **执行动作**: 行业增长趋势（近 12 个月口径） / 政策方向 / 海外制裁与出口管制清单 / 同业评级窗口
- **验证策略**: 行业数据用近 12 个月口径；制裁清单以官方文件为最终依据；研报观点冲突时优先采纳更新一期

**[[经营动态] 司法 / 舆情 / 合规]**
- **数据路由**: 司法公开数据 / 舆情聚合 / 监管处罚库 / 行业新闻
- **执行动作**: 近 24 个月被执行案件、重大诉讼、行政处罚（金额 + 处罚机关 + 整改进度）、突发负面事件
- **验证策略**: 媒体二次转载必须找到一手出处（监管公告 / 法院文书）；无法定位一手的标 "待补充"

**信源偏好**: 监管披露 > 评级机构 > 行业协会 > 行业研报 > 主流媒体 > 自媒体
**避坑指南**:
- 6 位数字可能是股票代码也可能是金额；遇到时先看上下文（"股票代码 600519" vs "存款 600519"）
- 同名企业（如多家"XX 科技")必须用统一社会信用代码或 ISIN 锁定
- "重整" ≠ "破产清算"，二者授信含义完全不同
- 港股 -W / -SW 后缀指 WVR / SVS 股权结构，估值口径不同

**数据裁决逻辑**:
1. 财报缺失 → 行业基准估算 + 明示假设，不编造
2. 多源冲突 → 监管披露 > 评级 > 财报 > 媒体
3. 时效冲突 → 取最新一期 + 标注信息时效窗口
4. 单源孤证 → 标"待复核"，不进结论
```

### `report_format`

```markdown
> **关键配置**
> - **目标对象**: [目标_授信_企业]
> - **报告类型**: 预尽调底稿
> - **基调设定**: 客观克制
> - **核心命题**: 用一份 30 分钟可读、可签字、可回查的底稿，帮客户经理把公开信息层的偿债/合规/经营风险摊开

# [目标_授信_企业] 对公授信预尽调底稿
报告时间: YYYY年MM月DD日

## 1. 关键发现与风险信号

> **[执行蓝图]**
> - **研究目标**: 在 200 字内让授信委一眼看见本次预尽调的最高优先级信号
> - **逻辑链条**: 动态识别 → 关键发现 → 最高优先级风险信号 → 关键不确定性 / 证据缺口
> - **信息需求**: 动态识别结果 / 关键发现 / 风险信号定级 / 不确定性自查清单
> - **输出形式**: 结论先行的四段式陈述

## 2. 主体身份核验

> **[执行蓝图]**
> - **研究目标**: 把公开身份信息核对到可签字一致
> - **逻辑链条**: 工商 / 披露 / 海外母公司 三处一致性检查
> - **信息需求**: 法人 / 注册资本 / 股权穿透 / 上市状态 / 审计意见 / 最新年报披露日
> - **输出形式**: 主体身份核验段落 + 关键差异说明

## 3. 行业地位与景气

> **[执行蓝图]**
> - **研究目标**: 定位目标企业在行业中的相对位置，并判断行业近 12 个月景气方向
> - **逻辑链条**: 行业景气与增长 → 监管政策 → 企业行业地位（份额或排名口径）→ 同业评级窗口
> - **信息需求**: 行业增速 / 主要政策事件 / 企业行业地位（份额或排名）/ 同业评级动向
> - **输出形式**: 行业景气与地位段落 + 政策事件时间线

## 4. 财务三年趋势（A 股 / 港股）或估算区间（非上市）

> **[执行蓝图]**
> - **研究目标**: 让授信委看到偿债能力的趋势线
> - **逻辑链条**: 营收 → 毛利 → 净利 → 经营性现金流 → 负债率 三年走势
> - **信息需求**: 近三个会计年度核心指标 + 异常波动归因
> - **输出形式**: 财务趋势段落（非上市企业给估算区间 + 明示假设）

## 5. 审计意见与评级窗口

> **[执行蓝图]**
> - **研究目标**: 审计意见保留 / 评级展望负面 是首批授信审批最直接的卡口
> - **逻辑链条**: 审计意见 → 评级展望 → 评级窗口变化
> - **信息需求**: 最新三期审计意见 / 评级机构 + 评级 + 展望 + 评级窗口
> - **输出形式**: 审计意见 + 评级摘录段落

## 6. 现金流与偿债结构

> **[执行蓝图]**
> - **研究目标**: 经营性现金流能否覆盖近 12 个月到期债务
> - **逻辑链条**: 经营性现金流 → 短期债务 → 利息覆盖倍数 → 再融资风险
> - **信息需求**: 经营性现金流 / 短期借款到期分布 / 利息覆盖倍数 / 在债余额
> - **输出形式**: 现金流-偿债匹配段落

## 7. 司法与被执行

> **[执行蓝图]**
> - **研究目标**: 近 24 个月被执行案件、重大诉讼对授信的影响
> - **逻辑链条**: 被执行案件 → 涉案金额 → 是否已结案 / 已和解
> - **信息需求**: 案件数 / 涉案金额 / 关键案件描述 / 是否影响授信担保资产
> - **输出形式**: 司法风险段落 + 关键案件清单

## 8. 监管处罚

> **[执行蓝图]**
> - **研究目标**: 近 36 个月行政处罚（环保 / 税务 / 安全生产 / 行业监管）
> - **逻辑链条**: 处罚事项 → 金额 → 整改进度 → 是否触发上市公司披露 / 评级下调
> - **信息需求**: 处罚机关 / 事由 / 金额 / 整改状态
> - **输出形式**: 监管处罚段落 + 关键案件清单

## 9. 海外制裁与出口管制（如适用）

> **[执行蓝图]**
> - **研究目标**: SEC / EU / OFAC 等海外监管清单覆盖
> - **逻辑链条**: 是否上海外制裁清单 → 直接 / 间接关联方 → 业务暴露口径
> - **信息需求**: 制裁清单状态 / 关联方 / 业务暴露
> - **输出形式**: 海外合规段落（无暴露写"经核 N 个清单未见命中"）

## 10. 舆情与突发事件

> **[执行蓝图]**
> - **研究目标**: 近 90 天突发负面事件 / 媒体关注度
> - **逻辑链条**: 一手出处 → 二手转载 → 是否影响经营 / 偿债
> - **信息需求**: 事件时间线 / 一手出处 / 持续影响判断
> - **输出形式**: 舆情时间线段落

## 11. 关联交易与担保（如可获）

> **[执行蓝图]**
> - **研究目标**: 关联方占款 / 对外担保 / 大额关联交易
> - **逻辑链条**: 关联方资金占用 → 关联担保规模 → 对偿债结构的影响
> - **信息需求**: 关联交易 / 担保余额 / 关联方信用状态
> - **输出形式**: 关联交易与担保段落

## 12. 自查与待核问题

> **[执行蓝图]**
> - **研究目标**: 列出本次预尽调发现的、需要正式尽调补足的核查项
> - **逻辑链条**: 信息缺失 → 风险归因 → 建议核查动作
> - **信息需求**: 待核条目 / 缺失原因 / 建议下一步动作
> - **输出形式**: 自查清单

## 13. 免责声明

> **[执行蓝图]**
> - **研究目标**: 标准化边界声明
> - **逻辑链条**: 信息来源 → 使用范围 → 时效说明
> - **信息需求**: 不替代正式尽调 / 不构成投资或授信决策建议
> - **输出形式**: 免责段落

---

> 本报告基于公开信息编制，不构成任何投资建议、授信决策或法律意见。
> 数据时效性以披露日为准；多源冲突已按 search_plan 中的裁决逻辑处理。
```

## 如何裁剪到你的场景

如果你的场景是金融/银行类相邻领域（如二级投研 / 财报点评 / 行业研究），保留架构骨架，按以下规则改：

1. **`input_form_spec`**: 把 `[目标_授信_企业]` 改成你的三段式变量
2. **`goal`**: 改写成简洁有力、价值优先的一段简介（解决什么问题 / 给什么价值，~40-80 字；不写角色代入、不硬码主体，详见 hard-rules R2）
3. **`search_plan`**: 4 个聚类换成你的数据源；维度数 2-5 之间
4. **`report_format`**: 章节数和顺序按你的报告结构改；保留蓝图四件套

每改一段，跑一次 `+validate <file>` 立刻发现违规。
